Participating Systems

  • Chainsaw (1.0, using FaCT++ 1.6.3)
    Chainsaw is a modular SROIQ reasoner for very large ontologies. It uses Atomic Decomposition and a pool of reasoners. To answer a certain query it choose a reasoner that contains a module suitable for that query and query that reasoner instead.
    License: LGPL v2
    Developers: Dmitry Tsarkov, Ignazio Palmisano
    Disciplines: OWL DL Classification, OWL EL Classification, OWL DL Consistency, OWL EL Consistency
  • ELepHant (0.3.3)
    ELepHant is a consequence-based reasoner for the OWL 2 EL fragment. Its aims are scalability to large ontologies and high-performance even on limited hardware. Currently supports a subset of the whole OWL2 EL profile and only has a command line interface that can parse ontologies in functional style syntax.
    License: Apache Licence 2.0
    Developer: Baris Sertkaya
    Disciplines: OWL EL Classification, OWL EL Consistency, OWL EL Realisation
  • ELK (0.5.0, unreleased)
    ELK is a fast reasoner for OWL 2 EL ontologies. Its distinctive feature is a parallel EL saturation procedure which concurrently computes super classes for each named class in the ontology.
    License: Apache Licence 2.0
    Developers: Yevgeny Kazakov, Markus Krötzsch, František Simančík, Pavel Klinov
    Disciplines:OWL EL Classification, OWL EL Consistency, OWL EL Realisation
  • FaCT++ (1.6.3)
    FaCT++ is the highly optimised tableaux-based SROIQ reasoner.
    License: LGPL v2
    Developer: Dmitry Tsarkov
    Disciplines: OWL DL Classification, OWL EL Classification, OWL DL Consistency, OWL EL Consistency, OWL DL Realisation, OWL EL Realisation
  • HermiT (1.3.8.1145)
    HermiT is a conformant OWL 2 DL reasoner that uses the direct semantics. It supports all OWL 2 DL constructs and the datatypes required by the OWL 2 specification. HermiT is based on the hypertableau calculus.
    License: LGPL 3.0
    Developers: Boris Motik, Rob Shearer, Birte Glimm, Giorgos Stoilos, and Ian Horrocks
    Disciplines:OWL DL Classification, OWL EL Classification, OWL DL Consistency, OWL EL Consistency, OWL DL Realisation, OWL EL Realisation
  • jcel (0.20.0)
    jcel is a reasoner for the description logic EL+ that uses a rule-based completion algorithm. jcel implements a subset of the OWL 2 EL profile, and can be used as a Java library or as a Protege plug-in.
    License: Apache Licence 2.0
    Developer: Julian Mendez
    Disciplines:OWL EL Classification, OWL EL Consistency
  • JFact (1.2.2)
    JFact is a Java port of the FaCT++ OWL DL reasoner with an improved datatype support.
    License: LGPL v2
    Developers: Dmitry Tsarkov, Ignazio Palmisano
    Disciplines: OWL DL Classification, OWL EL Classification, OWL DL Consistency, OWL EL Consistency, OWL DL Realisation, OWL EL Realisation
  • Konclude (0.5.3-386)
    Konclude is a parallel, high-performance reasoner for the Description Logic SROIQV(D). It is implemented in C++ and uses a reasoning technique that is based on a highly optimized tableau algorithm assisted by a completion-based saturation procedure.
    License: LGPL 3.0
    Developers: Andreas Steigmiller, Thorsten Liebig, Birte Glimm
    Disciplines: OWL DL Classification, OWL EL Classification, OWL DL Consistency, OWL EL Consistency, OWL DL Realisation, OWL EL Realisation
  • MORe (0.1.7)
    MORe exploits module extraction techniques to divide the workload of ontology reasoning between a reasoner for a lightweight fragment of OWL 2 and a fully fledged OWL 2 reasoner.
    License: LGPL 3.0
    Developers: Ana Armas Romero, Ernesto Jimenez Ruiz, Bernardo Cuenca Grau, Ian Horrocks, Cristina Feier
    Disciplines: OWL DL Classification, OWL EL Classification, OWL DL Consistency, OWL EL Consistency
  • TReasoner (1.0.1)
    TReasoner is a system for automated reasoning. It implements the classical tableau algorithm for the Description Logic SHOIQ with some optimisations and datatype support.
    License: LGPL v2
    Developer: Andrey Grigoryev
    Disciplines: OWL DL Classification, OWL EL Classification, OWL DL Consistency, OWL EL Consistency
  • TrOWL (1.4)
    TrOWL is a tractable reasoning infrastructure for OWL 2, supporting both TBox and ABox reasoning, as well as conjunctive query answering in SPARQL.
    License: Dual license
    For open source applications: AGPL
    For proprietary, closed source applications, and other commercial applications, TrOWL is available under alternative license terms.
    Developers: Jeff Z. Pan, Yuan Ren, Edward Thomas, Nophadol Jekjantuk, Stuart Taylor, Jhonatan Garcia
    Disciplines: OWL DL Classification, OWL EL Classification, OWL DL Realisation, OWL EL Realisation
